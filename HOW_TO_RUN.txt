===========================================
HOW TO RUN THE TRANSMISSION LINE FAULT DETECTION APP
===========================================

CHATBOT SETUP (GROQ API KEY)
----------------------------
Before starting the app, set your Groq API key in the same terminal:

PowerShell:
   $env:GROQ_API_KEY="PASTE_YOUR_GROQ_API_KEY_HERE"

Command Prompt:
   set GROQ_API_KEY=PASTE_YOUR_GROQ_API_KEY_HERE

Optional:
=
   set GROQ_MODEL=llama-3.1-8b-instant
   set GROQ_API_URL=https://api.groq.com/openai/v1/chat/completions

INTELLIGENCE CONSOLE SETUP (GEMINI API KEY)
-------------------------------------------
For the new intelligent response pipeline, set Gemini key in the same terminal:

PowerShell:
   $env:GEMINI_API_KEY="PASTE_YOUR_GEMINI_API_KEY_HERE"

Command Prompt:
   set GEMINI_API_KEY=PASTE_YOUR_GEMINI_API_KEY_HERE

Optional:

   set GEMINI_MODEL=gemini-2.0-flash
   set GEMINI_API_URL=https://generativelanguage.googleapis.com
   set GEMINI_MAX_RETRIES=3
   set GEMINI_FALLBACK_MODELS=gemini-2.0-flash,gemini-1.5-flash-latest,gemini-1.5-pro-latest

TROUBLESHOOTING:
----------------
If you see "Model not loaded":
- Make sure fault_model.joblib is in the same folder as app.py
- Check that you're using "py -3.14" (not just "py")

If you see "Port already in use":
- Another instance is running. Close it first or change the port in app.py

If the browser shows "This site can't be reached":
- Make sure the server started successfully
- Check that you're going to http://localhost:5000 (not https)
