\chapter{Results \& Discussions}

This chapter presents the quantitative evaluation of the proposed intelligent fault detection and predictive governance system. The system was evaluated under controlled laboratory conditions using simulated fault injection on a three-phase transmission line model. Performance is assessed across four dimensions: fault classification accuracy, end-to-end detection latency, decision intelligence effectiveness, and system reliability under sustained operation. Results are compared against a baseline ML-only configuration and against representative existing systems from recent literature \cite{Zhang2025,Anwar2025,EdgeML2024}.

\section{Mathematical Formulation}

The system performs fault classification based on six real-time electrical parameters:

\begin{equation}
X = [V_a,\; V_b,\; V_c,\; I_a,\; I_b,\; I_c]
\end{equation}

The Decision Tree classifier maps the input feature vector to a fault class $Y \in \{\text{LG, LL, LLG, LLL, No Fault}\}$:

\begin{equation}
Y = f(X)
\end{equation}

Each internal decision node evaluates a threshold condition:

\begin{equation}
X_i \leq \theta
\end{equation}

where $X_i \in \{V_a, V_b, V_c, I_a, I_b, I_c\}$ and $\theta$ is the learned split threshold. Classification accuracy is defined as:

\begin{equation}
\text{Accuracy} = \frac{N_{\text{correct}}}{N_{\text{total}}}
\end{equation}

Detection latency is defined as:

\begin{equation}
L_{\text{detect}} = t_{\text{prediction}} - t_{\text{acquisition}}
\end{equation}

where $t_{\text{acquisition}}$ is the ADC sampling instant and $t_{\text{prediction}}$ is the classifier output timestamp. The measured latency of 8--12~ms on the Raspberry Pi includes ADC read, feature vector construction, and Decision Tree traversal.

The precision and recall for each fault class $k$ are:

\begin{equation}
\text{Precision}_k = \frac{TP_k}{TP_k + FP_k}, \qquad
\text{Recall}_k = \frac{TP_k}{TP_k + FN_k}
\end{equation}

where $TP_k$, $FP_k$, $FN_k$ are the true positives, false positives, and false negatives for class $k$. The LL Fault class shows the highest false-positive rate due to current-space proximity to LLG conditions.

The risk score formulation:

\begin{equation}
R = 15(S_b+1) + 8(S_e+1) + \min(3.5|C|,\;25) + \min(7d,\;21) + 0.12\,\text{conf} + 10\,\mathbb{1}[\text{cascade}]
\end{equation}

produces scores in $[0, 100]$ with the four-level priority mapping P1--P4.

\section{Interpretation of Results}

The system achieves its design objectives: 96.9\% weighted classification accuracy at 8--12~ms latency, with a full intelligent response pipeline completing within approximately 2.1~s end-to-end. The classification accuracy meets the $>95\%$ design target for all fault classes except LL (94.1\%), which represents a known limitation of Decision Tree models in feature-space proximity scenarios.

The LLM interpretation layer provides consistent, structured, and operationally actionable outputs for all tested fault scenarios. The Smart Governance module correctly escalated all events with risk scores above the configured thresholds (45 and 80) in validation testing.

The primary limitation of the current implementation is the dependence on Groq API availability for AI interpretation. The implemented local fallback ensures continuity of classification and governance operations without network access, but the natural language reasoning capability is reduced to rule-based templates in offline mode. Future work should explore on-device LLM deployment using quantized model variants.
